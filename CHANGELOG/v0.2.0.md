Public-API rename: "structure factor" → "reflection data" throughout.

The renamed identifiers match crystallographic convention more accurately —
"reflection data" subsumes both post-refinement structure factors and the
integrated reflection data produced by processing pipelines (XDS, DIALS).

## Renames

The following public identifiers were renamed:

| v0.1.0                                          | v0.2.0                                              |
|-------------------------------------------------|-----------------------------------------------------|
| `FileType.CRYSTAL_STRUC_FACTORS`                | `FileType.CRYSTAL_REFLN_CIF`                        |
| `FileType.CRYSTAL_MTZ`                          | `FileType.CRYSTAL_REFLN_MTZ`                        |
| `Experiment(sf_only=...)`                       | `Experiment(refln_only=...)`                        |
| `DepositApi.create_ec_deposition(sf_only=...)`  | `DepositApi.create_ec_deposition(refln_only=...)`   |

Internally, the `_STRUCTURE_FACTOR_FILE_TYPES` set in `checks/file_checks.py`
was renamed to `_REFLN_FILE_TYPES`. The check-issue message for a missing
reflection data file now reads "Missing required reflection data file"
instead of "Missing required structure factors file".

## Wire-format compatibility

The OneDep server (per the upstream `wwPDB/py-onedep_deposition` client library)
continues to expect `sf_only` on the JSON wire for EC depositions. Within
`Experiment.json()` the renamed Python attribute is mapped back to `sf_only`
at the JSON-emit boundary, so v0.2.0 clients remain wire-compatible with
existing OneDep deployments. No coordinated server-side change is required.

## Deprecations

The old identifiers are retained as deprecation aliases. Using them emits
`DeprecationWarning` and resolves to the new name. They will be removed in v1.0.0.

To migrate, search-and-replace at your leisure:

| old                          | new                          |
|------------------------------|------------------------------|
| `CRYSTAL_STRUC_FACTORS`      | `CRYSTAL_REFLN_CIF`          |
| `CRYSTAL_MTZ`                | `CRYSTAL_REFLN_MTZ`          |
| `sf_only=`                   | `refln_only=`                |

To run your test suite with deprecations promoted to errors during the
migration window:

```bash
python -W error::DeprecationWarning -m pytest
```

## Breaking changes

None on the supported migration path. Code that used the v0.1.0 names will
continue to work and only emit `DeprecationWarning` until v1.0.0.
